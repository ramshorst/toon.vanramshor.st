## Context

This project started as a booking system for hotels, but from the beginning it was structured so it could be reused across multiple clients. The goal was not just to make one hotel website, but to create a deployable platform for the hospitality industry.

## What I built

I built the full product in PHP and MySQL on top of my own CMS. It included reservation management, customer records, editable email templates, a complex booking interface for viewing and managing reservations, and CRM-like tools for contacting guests. It also connected email communication back into IMAP folders so correspondence could be retrieved later.

## The technical challenge

The most interesting part was balancing hotel-specific needs with a reusable architecture. The system needed enough abstraction to be redeployed, while still supporting real operational workflows such as templated confirmations, reservation management, customer communication, and a calendar-oriented admin experience.

## Outcome

The project became an early SaaS-style system before I would have called it that. It pushed me further into application architecture, business workflow design, and building reusable products rather than one-off websites.
